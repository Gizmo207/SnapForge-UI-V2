# Audience

## Primary audience

A **single developer-operator** — the vault owner. They write and collect React
and HTML/CSS UI snippets and want a private, searchable, visual home for them
with frictionless capture and export. This is a single-user product by design:
the owner is the only account.

## Secondary actors (not accounts)

- **The owner's future self**, returning months later, relying on the gallery,
  classification, and search to re-find a component.
- **The export consumer** — whatever downstream project the exported zip bundle
  is dropped into. This actor never touches the app; it only receives bundles,
  which sets the bar for export cleanliness.

## Non-audience

- Teams or organizations needing shared, multi-user libraries.
- Consumers wanting a hosted, runnable component playground for others.
- Anyone needing the vault to execute or host arbitrary pasted code as a service.
